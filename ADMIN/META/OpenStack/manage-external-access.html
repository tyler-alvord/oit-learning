Managing Advanced External Network Access

Guest systems within OpenStack can be provided external network access for egress by associating it with a virtual router.

If access to the system is required from the external network (the lab), you can set this up for inbound external access by assigning an external ip address within the network settings in OpenStack

Openstack doesn't allow you to add an external IP addres directly to the host (as in, assigning an IP from the network to a Network Interface Card on the guest). Instead, you can leverage 1:1 Network Address Translation to provide external access to the Guests within Openstack. This feature is called "Floating IP"

Adding Floating IP addresses to the project

  1. Head to the "Network" tab within the "Project" menu
  2. Select the "Floating IPs" option from the resulting drop down
  3. Click on "Allocated IP to Project"
  4. Select the default pool - add a description if you'd like
  5. Done! The floating IP address has been added and can now be added to guests

Adding a floating IP to a virtual instance

  1. Head to the "Network" tab within the "Project" menu
  2. Select the "Floating IPs" option from the resulting drop down
  3. Next to the Allocated floating IP, select "Associate"
  4. Select the instance that you want to give the IP

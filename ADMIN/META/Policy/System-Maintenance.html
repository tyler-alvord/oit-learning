System Maintenance

Base system maintenance

Maintenance required for the host systems.
Maintenance Description & Notes Interval Manual or Automatic?
Software Updates Software should be regularly maintained. A simple apt-get upgrade should bring all relevant packages up to date to make sure that patches are applied to maintain basic system security. Weekly Automated
Operating system updates Ubuntu will need to be kept updated for security. Since OS updates can affect the functionality and stability, administrators should manually run the update commands. Since this system isn't critical to student functionality, there are not limitations to when in the day this can be done. Monthly Manual

OpenStack and Image Maintenance

Maintenance for the Openstack environment and the images that are contained within
Maintenance Description & Notes Interval Manual or Automatic?
System maintenance on environment images General maintenance (as described in the "Base System Maintenance" section of this document) should be run on the images that are used for production environments (infection monkey, sql, samba, ldap) for students. These are non-persistent images, so they should be maintained primarily to stay relevant for the users. Images that host applications (particularly security sensitive ones like LOIC) should be maintained with security patches to ensure that the application stays safe and operable. once per term (at the end of term- could be during break) Manual

Software updates

Software should be somewhat regularly maintained. A simple apt-get upgrade should bring all relevant packages up to date to make sure that patches are applied.

General Administrative Tasks

  • Image maintenance
  • Documentation maintenance
  • New Administrator Enablement

    • As students, it's likely that there will need to be handoffs either during busy seasons/times, or after student administrators graduate. Student administrators that know they will be unavailable for the administrative tasks of this system should notify the staff administrator of their absense with enought time to make the necessary adjustments - either selecting a new student administrator, or to notify the staff admin that they will need to keep an eye on the system for a duration.

    Ample student administrators should be kept so that the system can be generally maintained by students without excessive staff involvement. For the initial run of this system, a single student will probably suffice, but this may need to scale up as the system grows or as the administrative tasks become better understood.

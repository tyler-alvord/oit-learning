Admin Standards and Expectations Standards

Student Administrators

Student administrators are members of the student body that will be participating in the administration of the main systems involved in the learning environment - the controller and the compute node, and the images contained within the system.

General Administrative Tasks

  • Image maintenance
  • Documentation maintenance
  • New Administrator Enablement

    • As students, it's likely that there will need to be handoffs either during busy seasons/times, or after student administrators graduate. Student administrators that know they will be unavailable for the administrative tasks of this system should notify the staff administrator of their absense with enought time to make the necessary adjustments - either selecting a new student administrator, or to notify the staff admin that they will need to keep an eye on the system for a duration.

    Ample student administrators should be kept so that the system can be generally maintained by students without excessive staff involvement. For the initial run of this system, a single student will probably suffice, but this may need to scale up as the system grows or as the administrative tasks become better understood.

    Staff Administrators

    Administrative Structure

    Initially, there will only need to be one staff administrator at a time - to smooth out any future transitions, at least two staff members should be briefed on the functionality and maintenance of the system, as well as the process for selecting and training new administrators. The secondary staff admin doesn't need to fully understand the system, nor do they need to actually work on the system. They simply need to be briefed in order to help student administrators if something unforeseen affects the primary staff admin's ability to help with the system.

    Administrative Expectations

    Staff administrators (primary or secondary) should be the point of contact for other staff members, and should be able to provide insight into the available staff members that could fill the necessary roles. In the event that a staff member is leaving or otherwise becomming unable to fill the role of administrator, the other staff member should notify the student administrator(s) of the changes, and if possible, begin to facilitate the replacement staff administrator.

    Staff administrators' primary function is to facilitate student stewardship. Hands-on administration is an infrequent requirement for staff, but may be expected in times when student administrators are unavailable.

    Administrative Handoff

    1. In the event that a new student or staff administrator needs to be selected, the staff administrators should be the resource that provides a candidate list. This selection can occur unilaterally if the staff member has the time and inclination, or the list can be provided to the student administrators for final selection and onboarding.
    2. After administrative selection occurs, the student administrator can be the point of contact for providing the necessary resources to the new admin. This includes briefing on system functions, role expectations, and other involved parties.
      • Inform of and provide access to documentation (github)
      • Debrief on system functionality (images, components, etc)
    3. Student administrator are responsible for creating the account for the new administrator. A default password can be used if absolutely necessary on the condition that the new administrator changes the account password as soon as possible. Otherwise, the student administrator should create the account with the required permissions and allow the new administrator to set the password on their own.
      • Create an account for the administrator in the LDAP server
      • Create an account for the administrator in the OpenStack environment
      • Set permissions for the new user (add to sudoers, provide file permissions)
    4. If admin is leaving the organization, deprovision the previous administrator
      • Reset the password for LDAP
      • Reset the password for OpenStack
      • Delete/Deactivate account for LDAP
      • Delete/Deactivate account for OpenStack
    5. New adminstrator's tasks can be added to the "Contacts" table in this directory
